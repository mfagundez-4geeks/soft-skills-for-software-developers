# 01.6 练习：任务优先级排序

有效地优先排序任务对于顺利且高效的冲刺至关重要。组织冲刺待办事项列表的常用方法之一是 **MoSCoW 方法**，它帮助决定首先关注什么。

## 什么是 MoSCoW 方法？

MoSCoW 是一个首字母缩略词，代表：

- **Must**：必须在冲刺期间完成的关键任务。
- **Should**：如果可能应完成的重要任务，但可以推迟。
- **Could**：期望完成但跳过影响较小的任务。
- **Won't**：本次冲刺不会执行的任务。

该方法帮助团队优先考虑最高价值的功能，并管理对可推迟事项的期望。

## 使用 MoSCoW 的冲刺待办事项示例

想象一个为 Instagram 新功能“Stories”进行的冲刺。任务优先级可能是：

| 任务               | 描述                                   | 努力（小时） | 优先级 |
|--------------------|--------------------------------------|--------------|--------|
| UI 实现            | 设计和编码 Stories 的界面             | 12           | Must   |
| 后端 API           | 创建 Stories 数据的端点               | 15           | Must   |
| 添加滤镜           | 启用 Stories 中的照片滤镜             | 8            | Should |
| 分析跟踪           | 监控 Stories 的浏览和互动             | 5            | Could  |
| 多语言支持         | 添加本地化                           | 10           | Won't  |

## 如何优先排序冲刺任务

1. **列出所有任务**：写下所有计划的任务。
2. **分配 MoSCoW 优先级**：确定每个任务是 Must、Should、Could 还是 Won't。
3. **在每个优先级内排序任务**：从努力最大的 Must 任务开始，以避免最后一刻的混乱。
4. **使用工具**：使用 Jira 或 Trello 等项目管理工具标记和显示优先级。

## 练习：任务优先级排序

拿当前的开发任务列表，为每个任务分配 MoSCoW 优先级并估计所需努力。然后按 Must、Should、Could 的顺序排序列表。

# 01.6 練習：タスクの優先順位付け

タスクを効果的に優先順位付けすることは、スムーズで生産的なスプリントに不可欠です。スプリントバックログを整理するための一般的な方法の一つが**MoSCoW法**で、何に最初に集中すべきかを決めるのに役立ちます。

## MoSCoW法とは？

MoSCoWは以下の頭文字から成ります：

- **Must**：スプリント内で必ず完了しなければならない重要なタスク。
- **Should**：可能であれば行うべき重要なタスクだが、延期可能。
- **Could**：スキップしても影響が少ない、あったら良いタスク。
- **Won't**：このスプリントでは行わないタスク。

この方法は、チームが最も価値のある機能を優先的に提供し、何が後回しにできるかの期待値を管理するのに役立ちます。

## MoSCoWを使ったスプリントバックログの例

Instagramの新機能「ストーリーズ」のスプリントを想像してください。タスクの優先順位は以下のようになるかもしれません：

| タスク                  | 説明                              | 労力（時間） | 優先度   |
|-----------------------|---------------------------------|--------------|----------|
| UI実装                | ストーリーのインターフェース設計とコーディング | 12           | Must     |
| バックエンドAPI       | ストーリーデータのエンドポイント作成          | 15           | Must     |
| フィルター追加         | ストーリー内の写真フィルター有効化             | 8            | Should   |
| 分析トラッキング       | ストーリーの閲覧数とインタラクションの追跡    | 5            | Could    |
| 多言語対応             | ローカリゼーションの追加                       | 10           | Won't    |

## スプリントタスクの優先順位付け方法

1. **すべてのタスクをリストアップ**：作業予定のすべてのタスクを書き出します。
2. **MoSCoWの優先順位を割り当てる**：各タスクにMust、Should、Could、Won'tのいずれかを決定します。
3. **各優先度内でタスクを順序付ける**：Mustの中でも労力の大きいタスクから始め、直前の慌ただしさを避けます。
4. **ツールを活用する**：JiraやTrelloなどのプロジェクト管理ツールでタスクにタグを付け、優先順位を視覚化します。

## 練習：タスクの優先順位付け

現在の開発タスクのリストを取り出し、各タスクにMoSCoWの優先順位を割り当て、必要な労力を見積もってください。その後、Mustタスクを最優先に、次にShould、Couldの順にリストを並べ替えます。

### 理解度テスト

以下の各質問に正しい答えを選んでください。

1. スプリントで最初に完了すべきMoSCoWの優先順位はどれですか？

   - [ ] Could
   - [ ] Should
   - [x] Must
   - [ ] Won't

2. 「Won't」の優先順位は何を意味しますか？

   - [ ] 今すぐ完了しなければならない重要なタスク
   - [ ] 可能であれば行うべきタスク
   - [ ] あったら良いが影響が少ないタスク
   - [x] このスプリントでは行わないタスク

3. なぜMustカテゴリー内で労力の大きいタスクを早めに順序付けることが有効ですか？

   - [ ] 簡単なタスクを先に終わらせるため
   - [x] 直前の慌ただしさを避け、作業負荷を分散するため
   - [ ] 労力は重要でないため
   - [ ] 重要な作業を遅らせるため

4. JiraやTrelloのようなツールはMoSCoWの優先順位付けにどのように役立ちますか？

   - [ ] タスクを自動的に完了させる
   - [x] 優先順位ごとにタスクを視覚化しタグ付けするのに役立つ
   - [ ] 優先順位付けの必要性をなくす
   - [ ] バグのみを追跡する

この方法を練習することで、スプリント計画のスキルが向上し、チームが効率的に価値を提供できるようになります！

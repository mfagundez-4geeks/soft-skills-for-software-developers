# 01.6 Práctica: Priorización de Tareas

Priorizar tareas de manera efectiva es esencial para un sprint fluido y productivo. Un método popular para organizar tu backlog de sprint es el **método MoSCoW**, que te ayuda a decidir en qué enfocarte primero.

## ¿Qué es el método MoSCoW?

MoSCoW significa:

- **Must**: Tareas críticas que deben completarse en el sprint.
- **Should**: Tareas importantes que deberían hacerse si es posible, pero pueden posponerse.
- **Could**: Tareas deseables con bajo impacto si se omiten.
- **Won't**: Tareas que no se harán en este sprint.

Este método ayuda a los equipos a enfocarse en entregar primero las características más valiosas mientras gestionan las expectativas sobre lo que puede esperar.

## Ejemplo de Backlog de Sprint usando MoSCoW

Imagina un sprint para la nueva función "Stories" de Instagram. Así podrían priorizarse las tareas:

| Tarea                 | Descripción                        | Esfuerzo (hrs) | Prioridad |
|-----------------------|----------------------------------|----------------|-----------|
| Implementar UI        | Diseñar y codificar la interfaz de historias | 12             | Must      |
| API Backend           | Crear endpoints para datos de historias | 15             | Must      |
| Añadir filtros        | Habilitar filtros de fotos en historias | 8              | Should    |
| Seguimiento analítico | Rastrear vistas e interacciones de historias | 5              | Could     |
| Soporte multilingüe   | Añadir localización               | 10             | Won't     |

## Cómo priorizar tus tareas de sprint

1. **Lista todas las tareas**: Anota cada tarea en la que planeas trabajar.
2. **Asigna prioridades MoSCoW**: Decide qué tareas son Must, Should, Could o Won't.
3. **Ordena las tareas dentro de cada prioridad**: Comienza con las tareas Must de mayor esfuerzo para evitar prisas de último minuto.
4. **Usa herramientas**: Etiqueta las tareas en Jira, Trello o tu herramienta de gestión de proyectos preferida para visualizar prioridades.

## Práctica: Prioriza tus tareas

Toma tu lista actual de tareas de desarrollo. Asigna a cada una una prioridad MoSCoW y estima el esfuerzo requerido. Luego, reordena tu lista para enfocarte primero en las tareas Must, seguidas de Should y Could.

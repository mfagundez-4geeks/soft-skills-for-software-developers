# 01.4 练习：团队协作场景

在一个产品团队会议中，讨论是否在类似Instagram界面的应用侧边栏添加新功能。一位开发者支持此举以增加用户参与度，而另一位则担心视觉复杂性。团队如何建设性地克服这种意见分歧？

## 第一步：积极倾听与复述
首先，积极倾听对方的意见并复述以确认理解。例如：

> “你的意思是你担心添加这个功能会让用户感到负担，界面变得不那么易用？”

这种方法尊重对方观点，减少防御性态度。

## 第二步：清晰且礼貌地表达自己的观点
接着，清晰且礼貌地表达自己的观点：

> “我理解保持界面简洁的重要性，但添加这个功能可以提升内容发现，增加用户粘性。”

## 第三步：寻找共识并提出折中方案
最后，找到兼顾双方关切的解决方案：

> “你们觉得做一个最小可行版本的原型，先测试用户反应如何，再决定是否全面实施怎么样？”

这个流程——**倾听、复述、表达、提出折中**——帮助团队克服冲突，实现协作。

## 实践练习：编写对话脚本
练习时，尝试扮演双方角色，按照以下模板进行简短对话：

```markdown
**开发者A：** [复述开发者B的担忧]

**开发者B：** [确认或解释]

**开发者A：** [礼貌表达自己的观点]

**开发者B：** [回应]

**开发者A：** [提出折中方案或下一步]

**开发者B：** [接受或提出调整]
```

此技巧在Twitter等公司真实会议中也很有用，能平衡创新与可用性。

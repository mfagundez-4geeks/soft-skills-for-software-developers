# 01.4 Practice: Team Collaboration Scenario

Imagine a product team meeting debating whether to add a new feature to an app's sidebar, similar to Instagram's interface. One developer supports it for user engagement; another worries about clutter. How can the team navigate this disagreement constructively?

## Step 1: Active Listening and Paraphrasing
Start by **actively listening** to the opposing view and paraphrasing it to confirm understanding. For example:

> "So you’re concerned that adding this feature might overwhelm users and reduce usability, is that right?"

This approach validates the other person's perspective and lowers defensiveness.

## Step 2: Express Your Viewpoint Clearly and Respectfully
Next, share your own viewpoint using clear but respectful language:

> "I see the value in keeping the interface clean, but adding this feature could increase user retention by making content more discoverable."

## Step 3: Seek Common Ground and Propose a Compromise
Finally, look for a solution that addresses both concerns:

> "Maybe we can prototype a minimal version and test user reactions before fully committing."

This pattern—**listen, paraphrase, express, and propose compromise**—helps teams move past conflict toward collaboration.

## Practice Activity: Script Your Dialogue
Try applying this by scripting a short dialogue where you play both roles, practicing these steps. Use the template below as a guide:

```markdown
**Developer A:** [Paraphrase Developer B's concern]

**Developer B:** [Confirm or clarify]

**Developer A:** [Express own viewpoint respectfully]

**Developer B:** [Respond]

**Developer A:** [Propose a compromise or next step]

**Developer B:** [Agree or suggest adjustments]
```

This technique mirrors real-world meetings at companies like Twitter, where balancing innovation and usability is key.

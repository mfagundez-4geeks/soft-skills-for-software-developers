# 01.1 コミュニケーションの基本

効果的なコミュニケーションは、成功するソフトウェア開発チームの基盤です。身につけるべき最も強力なコミュニケーションスキルの一つが**アクティブリスニング**です。これは話し手に完全に集中し、彼らのメッセージを理解し、思慮深く応答することを意味します。

## アクティブリスニングの習得

開発者環境でアクティブリスナーになるための重要な技術は以下の通りです：

1. **気を散らすものを排除する。** 誰かが話しているときは、携帯電話をしまい、関係のないタブを閉じ、全注意を向けましょう。例えば、チームミーティング中はノートパソコンを閉じ、目を合わせて関与を示します。

2. **非言語的な合図を使う。** 頷く、少し前かがみになる、笑顔を見せることで、注意を払っていることを示し、話し手を促します。

3. **理解を確認するために言い換える。** チームメイトがバグを説明した後、「つまり、アプリがユーザーデータを取得しようとするときに問題が起きるということですね？」と言うことで、彼らの意見を尊重し誤解を避けます。

4. **オープンエンドの質問をする。** 「バグは直しましたか？」のようなはい/いいえで答えられる質問の代わりに、「バグを解決するためにどんな方法を試しましたか？」と尋ねることで、詳しい説明や深い議論を促します。

5. **話を遮らない。** 話し手が考えを終えるまで待ちましょう。ポイントを覚えておく必要がある場合は、口を挟む代わりにメモを取ります。

6. **感情と内容を反映する。** 「デプロイの遅れに苛立っているようですね」といった発言は共感と信頼を築きます。

## なぜ開発者にとってアクティブリスニングが重要か

アクティブリスニングは明確さを高め、ミスを減らし、コードレビュー、スプリント計画、日々のスタンドアップミーティングでのチームワークを強化します。複雑な技術的問題を理解し、技術者と非技術者の両方と効果的に協力するのに役立ちます。

### アクティブリスニング技術の理解を試す穴埋め問題：

```fill_in_the_blank 1="気を散らすものを排除" 2="言い換える" 3="オープンエンド" 4="話を遮らない" 5="感情を反映する"

アクティブリスニングを実践するには、まず携帯電話をしまい関係のないタブを閉じて_1_ことが必要です。

誰かが問題を説明した後、そのメッセージを_2_して正しく理解したか確認できます。

_3_質問をすることで、話し手がより詳細な情報を提供するよう促します。

話し手が考えを終えるまで_4_ことが重要です。

最後に、会話中に共感を示し信頼を築くために_5_ことが必要です。
```

これらの技術を練習することで、コミュニケーションスキルが向上し、より効果的で調和のとれた開発チームに貢献できます。

# 01.1 沟通基础

有效的沟通是成功软件开发团队的基础。掌握的最强大沟通技能之一是**积极倾听**。这意味着完全专注于对话者，理解其信息并做出深思熟虑的回应。

## 学习积极倾听

在开发环境中成为积极倾听者的关键技巧如下：

1. **消除干扰。** 当有人讲话时，收起手机，关闭无关标签页，集中全部注意力。例如，在团队会议中，关闭笔记本电脑并保持眼神交流以显示参与感。

2. **使用非语言信号。** 点头，身体微微前倾，微笑以示关注并鼓励对话者。

3. **复述以确认理解。** 当同事解释一个错误后，说“所以，问题发生在应用尝试获取用户数据时，对吗？”显示对其观点的尊重并避免误解。

4. **提出开放性问题。** 避免“你解决了这个错误吗？”这类是非题，改问“你尝试了哪些方法来解决这个错误？”以激发详细解释和深入讨论。

5. **不要打断。** 等待对话者表达完想法。如果需要记住某点，做笔记而非打断。

6. **反映情感和内容。** 诸如“你似乎对部署延迟感到沮丧”这类句子建立同理心和信任。

## 为什么积极倾听对开发者很重要

积极倾听提高了清晰度，减少错误，增强代码审查、冲刺计划和每日会议中的团队合作。它有助于理解复杂技术问题，并与技术和非技术人员有效协作。

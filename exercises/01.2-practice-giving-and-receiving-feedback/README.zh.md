# 01.2 练习：如何给予和接受反馈

有效的反馈是高效软件开发团队的基础。它有助于提高代码质量，建立信任，并支持持续学习。在本课中，我们将学习如何使用 **SBI（情境-行为-影响）** 模型和 **积极倾听** 技巧，建设性地给予和接受反馈。

## 结构化反馈的SBI模型

SBI模型帮助将反馈分为三部分，传达清晰、尊重且可执行的反馈：

- **情境（Situation）：** 描述行为发生的具体情境。
- **行为（Behavior）：** 描述可观察的行为，不带评判。
- **影响（Impact）：** 分享该行为对你、团队或项目产生的影响。

### 示例：

“在最近的登录功能拉取请求中（情境），我注意到错误处理没有覆盖网络故障（行为）。这可能导致应用静默失败时用户感到困惑（影响）。”

使用第一人称“我”而非“你”的表达，有助于避免指责，保持对话积极。例如，说“我认为这里增加错误处理会提升用户体验”，而不是“你忘了处理错误”。

## 通过积极倾听接受反馈

当你接受反馈时，遵循以下步骤：

- **在回应前先停顿。** 给自己时间消化反馈。
- **复述你听到的内容。** 例如，“所以你的意思是错误处理应该更健壮，对吗？”
- **提出澄清问题。** 例如，“你能建议如何改进网络错误处理吗？”
- **以建设性方式回应。** 感谢反馈者，避免防御性反应。

## 实践场景：GitHub拉取请求评论

想象你正在审查GitHub上的一个拉取请求。以下是反馈交流示例：

```text
审查者：“在最近的登录功能PR中，我注意到错误处理没有覆盖网络故障。这可能导致应用静默失败时用户感到困惑。”
开发者：“感谢指出。我会添加更全面的错误处理。”
```

尝试使用SBI模型和第一人称信息重写审查者的评论，并用积极倾听创建建设性回复。

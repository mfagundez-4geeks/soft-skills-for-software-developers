# 01.2 Übung: Wie man Feedback gibt und erhält

Effektives Feedback ist die Grundlage für ein produktives Softwareentwicklungsteam. Es hilft, die Codequalität zu verbessern, Vertrauen aufzubauen und kontinuierliches Lernen zu unterstützen. In dieser Lektion lernen wir, wie man konstruktiv Feedback gibt und erhält, indem wir das **SBI-Modell (Situation-Verhalten-Auswirkungen)** und Techniken des **aktiven Zuhörens** verwenden.

## SBI-Modell für strukturiertes Feedback

Das SBI-Modell hilft, klares, respektvolles und umsetzbares Feedback zu kommunizieren, indem es in drei Teile gegliedert wird:

- **Situation:** Beschreiben Sie die spezifische Situation, in der das Verhalten aufgetreten ist.
- **Verhalten:** Beschreiben Sie das beobachtbare Verhalten ohne Wertung.
- **Auswirkungen:** Teilen Sie mit, welche Auswirkungen dieses Verhalten auf Sie, das Team oder das Projekt hatte.

### Beispiel:

"Im kürzlichen Pull Request der Login-Funktionalität (Situation) habe ich bemerkt, dass die Fehlerbehandlung Netzwerkausfälle nicht abdeckt (Verhalten). Dies kann zu Verwirrung bei den Nutzern führen, wenn die App stillschweigend fehlschlägt (Auswirkungen)."

Die Verwendung von Ich-Botschaften **"ich"** anstelle von "du" hilft, Vorwürfe zu vermeiden und das Gespräch positiv zu halten. Zum Beispiel zu sagen: "Ich denke, dass das Hinzufügen einer Fehlerbehandlung hier die Benutzererfahrung verbessern würde" anstatt "Du hast die Fehlerbehandlung vergessen".

## Feedback erhalten mit aktivem Zuhören

Wenn Sie Feedback erhalten, befolgen Sie diese Schritte:

- **Halten Sie inne, bevor Sie antworten.** Nehmen Sie sich Zeit, um das Feedback zu verarbeiten.
- **Paraphrasieren Sie, was Sie gehört haben.** Zum Beispiel: "Also sagen Sie, dass die Fehlerbehandlung robuster sein sollte, richtig?"
- **Stellen Sie klärende Fragen.** Zum Beispiel: "Könnten Sie vorschlagen, wie die Fehlerbehandlung bei Netzwerkausfällen verbessert werden kann?"
- **Antworten Sie konstruktiv.** Danken Sie der Person, die Ihnen Feedback gegeben hat, und vermeiden Sie defensive Reaktionen.

## Praktisches Szenario: Kommentare zu einem GitHub Pull Request

Stellen Sie sich vor, Sie überprüfen einen Pull Request auf GitHub. Hier ist ein Beispiel für einen Feedback-Austausch:

```text
Reviewer: "Im kürzlichen PR der Login-Funktionalität habe ich bemerkt, dass die Fehlerbehandlung Netzwerkausfälle nicht abdeckt. Dies kann zu Verwirrung bei den Nutzern führen, wenn die App stillschweigend fehlschlägt."
Entwickler: "Danke für den Hinweis. Ich werde eine umfassendere Fehlerbehandlung hinzufügen."
```

Versuchen Sie, den Kommentar des Reviewers mit dem SBI-Modell und Ich-Botschaften umzuschreiben und eine konstruktive Antwort mit aktivem Zuhören zu formulieren.

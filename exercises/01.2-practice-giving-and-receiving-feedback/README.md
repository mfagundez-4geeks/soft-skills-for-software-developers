# 01.2 Practice: Giving and Receiving Feedback

Effective feedback is a cornerstone of productive software development teams. It helps improve code quality, fosters trust, and supports continuous learning. In this lesson, you'll learn how to give and receive feedback constructively using the **SBI (Situation-Behavior-Impact) model** and **active listening** techniques.

## The SBI Model for Structured Feedback

The SBI model helps you deliver clear, respectful, and actionable feedback by breaking it down into three parts:

- **Situation:** Describe the specific context where the behavior occurred.
- **Behavior:** Explain the observable action without judgment.
- **Impact:** Share the effect this behavior had on you, the team, or the project.

### Example:

"In the recent login feature pull request (Situation), I noticed the error handling doesn't cover network failures (Behavior). This could cause user confusion if the app silently fails (Impact)."

Using **"I" statements** instead of "you" helps avoid blame and keeps the conversation positive. For example, say, "I think adding error handling here would improve user experience," rather than, "You forgot error handling."

## Receiving Feedback with Active Listening

When receiving feedback, practice these steps:

- **Pause before responding.** Take a moment to process the feedback.
- **Paraphrase what you heard.** For example, "So you're saying the error handling could be more robust?"
- **Ask clarifying questions.** Such as, "Can you suggest how to handle network errors better?"
- **Respond constructively.** Thank the giver and avoid defensive reactions.

## Practice Scenario: GitHub Pull Request Comment

Imagine reviewing a pull request on GitHub. Here's a sample feedback exchange:

```text
Reviewer: "In the recent login feature PR, I noticed the error handling doesn't cover network failures. This could cause user confusion if the app silently fails."
Developer: "Thanks for pointing that out. I'll add more comprehensive error handling."
```

Try rewriting the reviewer's comment using the SBI model and "I" statements, then draft a constructive response using active listening.

# 01.8 クイズ：ソフトスキルの知識をテストしよう

これらの実際のシナリオであなたのソフトスキルを試してみましょう！各状況を注意深く読み、ソフトウェア開発環境で効果的なソフトスキルを示す最適な対応を選んでください。

1. チームメイトが一貫して締め切りを守らず、プロジェクトに遅れが生じています。どのように対処すべきでしょうか？

   - [ ] 問題を無視して自然に改善するのを待つ。

   - [ ] 次のチームミーティングで公にチームメイトを非難する。

   - [x] 個別に締め切りの遅れについて話し、「私」メッセージを使って懸念を伝え、改善のためのサポートを提供する。

   - [ ] 誰にも知らせずに彼らのタスクを引き継ぎ、プロジェクトを軌道に乗せる。


2. コードレビュー中に、同僚が建設的なフィードバックを受けて防御的になりました。最善の対応は何でしょうか？

   - [ ] フィードバックを議論なしで受け入れるよう強要する。

   - [x] 彼らの感情を認め、フィードバックはコードに対するものであって人に対するものではないことを明確にし、協力して改善方法を提案する。

   - [ ] さらなるフィードバックを避けて対立を防ぐ。

   - [ ] プロ意識を促すために態度を批判する。


3. チームミーティングで、2人のメンバーがプロジェクトのアプローチについて対立しています。この状況で使うべきソフトスキルは何でしょうか？

   - [ ] 意見の不一致を無視して自分のアイデアを進める。

   - [ ] 声の大きいメンバーの味方をする。

   - [x] 敬意を持った議論を促進し、アイデアの共有を奨励し、チームが妥協点を見つけるのを助ける。

   - [ ] 対立を避けるために決定を無期限に延期する。


4. 締め切りが厳しい複数のタスクがあります。効果的に時間を管理するにはどうしますか？

   - [ ] 思いつくままにランダムにタスクに取り組む。

   - [ ] 簡単なタスクだけに集中して早く終わらせる。

   - [x] 緊急度と影響に基づいてタスクの優先順位をつけ、スケジュールを作成し、チームに進捗を伝える。

   - [ ] 他の人にすべてのタスクを任せ、彼らの負荷を確認しない。


頑張ってください！選択を振り返り、これらのソフトスキルが開発者としての日常業務にどのように適用されるか考えてみましょう。

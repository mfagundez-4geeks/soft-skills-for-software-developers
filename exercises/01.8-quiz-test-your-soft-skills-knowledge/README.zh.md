# 01.8 测验：测试你的软技能知识

通过这些真实场景测试你的软技能！仔细阅读每个情境，选择在软件开发环境中展示有效软技能的最佳答案。

1. 团队成员经常未能按时完成任务，导致项目延误。你应该如何处理这种情况？

   - [ ] 忽视问题，等待其自然改善。

   - [ ] 在下一次会议上公开批评该团队成员。

   - [x] 私下与他谈论延误问题，使用第一人称表达关切，并提供支持以帮助改进。

   - [ ] 在不通知任何人的情况下承担他的任务，以使项目回到正轨。


2. 在代码审查中，一位同事在收到建设性反馈后变得防御性。最佳回应是什么？

   - [ ] 坚持让他无条件接受反馈。

   - [x] 认可他的感受，澄清反馈针对代码而非个人，并合作寻找改进方法。

   - [ ] 避免进一步反馈以防止冲突。

   - [ ] 批评他的态度以促进专业精神。


3. 在团队会议中，两名成员对项目方法存在分歧。你应该使用哪种软技能？

   - [ ] 忽视分歧，按自己的想法进行。

   - [ ] 支持声音更大的一方。

   - [x] 促进尊重的讨论，鼓励分享想法，帮助团队找到折中方案。

   - [ ] 无期限推迟决策以避免冲突。


4. 你有多个任务且截止日期临近。如何有效管理时间？

   - [ ] 随意处理任务。

   - [ ] 只专注于简单任务以快速完成。

   - [x] 根据紧急性和影响优先排序任务，制定计划并向团队汇报进展。

   - [ ] 将所有任务委派给他人而不检查他们的工作负载。


祝你好运！反思你的选择，并考虑这些软技能如何应用于你作为开发者的日常工作。

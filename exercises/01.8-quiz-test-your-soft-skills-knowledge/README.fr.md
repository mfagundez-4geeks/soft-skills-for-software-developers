# 01.8 Quiz : Testez vos connaissances sur les compétences comportementales

Testez vos compétences comportementales avec ces scénarios réels ! Lisez attentivement chaque situation et choisissez la meilleure réponse qui démontre des compétences comportementales efficaces dans un environnement de développement logiciel.

1. Un membre de l'équipe ne respecte pas constamment les délais, causant des retards dans le projet. Comment devriez-vous gérer la situation ?

   - [ ] Ignorer le problème et attendre qu'il s'améliore naturellement.

   - [ ] Critiquer publiquement le membre de l'équipe lors de la prochaine réunion.

   - [x] Parler en privé avec lui concernant les retards, en utilisant des messages à la première personne pour exprimer les préoccupations et offrir un soutien pour s'améliorer.

   - [ ] Prendre en charge ses tâches sans en informer personne pour remettre le projet sur les rails.


2. Lors d'une revue de code, un collègue devient sur la défensive après avoir reçu des retours constructifs. Quelle est la meilleure réponse ?

   - [ ] Insister pour qu'il accepte les retours sans discussion.

   - [x] Reconnaître ses sentiments, clarifier que les retours concernent le code et non la personne, et collaborer pour trouver des moyens d'amélioration.

   - [ ] Éviter d'autres retours pour prévenir les conflits.

   - [ ] Critiquer son attitude pour promouvoir le professionnalisme.


3. Lors d'une réunion d'équipe, deux membres sont en désaccord sur l'approche du projet. Quelle compétence comportementale devriez-vous utiliser dans cette situation ?

   - [ ] Ignorer le désaccord et poursuivre avec votre idée.

   - [ ] Prendre parti pour le membre le plus vocal.

   - [x] Faciliter une discussion respectueuse, encourager le partage des idées et aider l'équipe à trouver un compromis.

   - [ ] Reporter indéfiniment la décision pour éviter les conflits.


4. Vous avez plusieurs tâches avec des échéances rapprochées. Comment gérez-vous efficacement votre temps ?

   - [ ] Aborder les tâches de manière aléatoire.

   - [ ] Vous concentrer uniquement sur les tâches les plus simples pour les terminer rapidement.

   - [x] Prioriser les tâches en fonction de l'urgence et de l'impact, créer un planning et communiquer les progrès à l'équipe.

   - [ ] Déléguer toutes les tâches à d'autres sans vérifier leur charge de travail.


Bonne chance ! Réfléchissez à vos choix et considérez comment ces compétences comportementales s'appliquent à vos activités quotidiennes en tant que développeur.
